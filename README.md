# Мониторинг позиции абитуриента в конкурсных списках

Каждый день ищет абитуриента по **единому коду с Госуслуг (УИД = `1212030`)**
во всех конкурсных группах пяти вузов и присылает отчёт в Telegram:

> вуз · код · специальность · **сырая позиция** · **реальная позиция** · бюджетных мест

«Реальная позиция» — оценка: среди тех, кто выше по списку, учитываются только
реальные конкуренты (подали согласие на эту группу **или** она у них в
приоритете №1). Остальные выше, скорее всего, уйдут в другой вуз/на другой
приоритет и освободят строчку. Точная картина требует расчёта по всем вузам
сразу, поэтому это приближение — но обычно близкое к правде.

## 1. Быстрый старт (локально)

```bash
pip install -r requirements.txt
python -m playwright install chromium     # для ЛЭТИ
export APPLICANT_ID=1212030
python abitur_monitor.py
```

Скрипт напечатает таблицу в консоль и сохранит `report.csv`.
Волгатех должен найтись сразу.

## 2. Telegram

1. В Telegram напишите **@BotFather** → `/newbot` → получите **токен**.
2. Узнайте свой **chat_id**: напишите боту любое сообщение, затем откройте
   `https://api.telegram.org/bot<ТОКЕН>/getUpdates` и найдите `"chat":{"id":...}`.
3. Задайте переменные и запустите:

```bash
export TG_TOKEN="123456:AA..."
export TG_CHAT_ID="987654321"
python abitur_monitor.py
```

## 3. Ежедневный запуск (бесплатно, GitHub Actions)

1. Создайте репозиторий на GitHub, загрузите туда эти файлы.
2. **Settings → Secrets and variables → Actions → New repository secret**, добавьте:
   - `TG_TOKEN`, `TG_CHAT_ID`
   - (позже) `LETI_GROUPS`, при желании `BONCH_ENDPOINT`, `BONCH_GROUPS`
3. Готово: `.github/workflows/daily.yml` запускает скрипт каждый день в 08:00 МСК.
   Кнопка ручного запуска — во вкладке **Actions → abitur-monitor → Run workflow**.

## 4. Бонч — разобрать AJAX (один раз)

Список на `priem.sut.ru/spisok-abiturientov` подгружается отдельным запросом.

1. Откройте страницу, выберите уровень/форму/основу/группу так, чтобы список
   появился.
2. **F12 → вкладка Network → Fetch/XHR**, повторите выбор группы.
3. Найдите запрос, в ответе которого видна таблица абитуриентов. Скопируйте его
   **URL** и **параметры** (а также id конкурсных групп).
4. Впишите в секреты:
   - `BONCH_ENDPOINT` — URL запроса
   - `BONCH_GROUPS` — JSON-массив id групп, напр. `["123","124"]`
5. При необходимости поправьте имена полей в `_bonch_from_json()` под реальный
   ответ (там сейчас типовая заготовка).

## 5. ЛЭТИ — собрать id групп (один раз)

Сайт — SPA и закрыт для роботов, поэтому используется headless-браузер.
Конкурсные списки появятся **27 июля 2026**. Нужно сопоставить ваши коды с id групп:

1. Откройте список нужной программы на `abit.etu.ru`. В адресной строке будет
   `…/lists/page/#/?id=<GUID>`. Скопируйте `<GUID>`.
2. Повторите для каждого из кодов: `09.03.01`, `09.03.02`, `10.05.01`, `27.03.03`
   (у программы может быть несколько групп — основная/квоты; берите бюджетную
   «основные места»).
3. Соберите карту и положите в секрет `LETI_GROUPS`:

```json
{
  "09.03.01": ["019ee529-454f-7e45-aced-7f2361797e11"],
  "09.03.02": ["..."],
  "10.05.01": ["..."],
  "27.03.03": ["..."]
}
```

(Пример уже содержит id из вашей ссылки — подставьте его к нужному коду.)

## 6. (Опционально) запись в вашу Google-таблицу

1. В Google Cloud создайте **сервис-аккаунт**, скачайте `service_account.json`.
2. Дайте этому сервис-аккаунту (его email вида `...@...iam.gserviceaccount.com`)
   доступ **Редактор** к вашей таблице.
3. Запускайте с `SHEETS_ENABLED=1` и положите `service_account.json` рядом.
   Результат попадёт на вкладку «Результаты».

## Важно про сроки

Прямо сейчас данные реально отдают **Волгатех, ГУАП и Бонч**. Рейтинги **СПбГУ**
и **ЛЭТИ** открываются в конце июля — скрипт подключит их сам, как только списки
выйдут (для ЛЭТИ нужно лишь один раз вписать id групп).

## Если что-то не нашлось

- На первом запуске сверьте `report.csv` с тем, что видите на сайте глазами.
- Если вуз отдаёт пустоту — скорее всего, его список ещё не опубликован
  (см. сроки) или у него изменилась вёрстка; загляните в лог (он подробный).
